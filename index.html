origin

Born from
coffee & laziness.

I got tired of putting my hand back on the touchpad every time I needed to turn a page — while my hands were busy doing something more important, like holding coffee.

So instead of accepting that reality, I decided to build a mouse you wear on your finger. A ring with an outer rotary band for scrolling, an IMU for pointer movement, and enough stubbornness to actually make it happen.

This is a FOSS hardware project. If it ever reaches production, profit isn't the point. The point is that a thing like this should exist and currently doesn't.

The hardware
inside the ring.

01 // BRAIN
ESP32-C3FH4 / ESP8684-H2
RISC-V core chosen as much for ideology as for size. Fits the critically small inner ring, handles BLE HID, runs FreeRTOS. Possibly the smallest ESP32 you'll ever hold.
RISC-V // BLE
02 // MOTION
ICM-20948
9-DoF IMU — accelerometer, gyroscope, magnetometer in one tiny package. Excellent docs, available dev board for firmware prototyping. Replaced BMA423 + LSM303 combo.
9-DOF // SPI // I2C
03 // BIOMETRICS
MAX30102
Pulse oximeter & heart-rate sensor. Optional — toggleable by the user for privacy and battery conservation. Because biometric data is yours, not anyone else's.
OPT-IN // HR
04 // SCROLL
Rotary Outer Ring + Hall Sensor
The outer ring spins mechanically. Hall sensor reads magnetic pulses to detect direction and velocity. Same rotation could theoretically contribute to inductive charging — TBD.
HALL // SCROLL
05 // FORM
FlexPCB + OpenSCAD Body
Designed for FlexPCB in EasyEDA Pro. 3D model via Jupyter + SolidPython + JupyterSCAD. RX/TX probes exposed for FOSS hackability. This thing is meant to be tinkered with.
FLEXPCB // FOSS
06 // POWER
Battery — TBD
Small LiFePO cell (~22mAh, similar to Oura Ring research). Thermoelectric and rotational inductive charging are ideas on the table. Nothing is settled.
TBD // LIFEPO

Software
stack.

Arduino IDE + ICM20948_WE
Working base. SPI abstraction layer by wollewald. Comfortable enough to iterate fast.
FreeRTOS tasks
Separate tasks for comms, IMU data, status LED. Event-driven, threshold-triggered BLE sends.
Gamepad → Mouse HID
IMU 3D data projected to 2D plane, routed through BLE HID compatibility layer to mouse events.
JupyterSCAD + SolidPython
Parametric 3D ring design. Jupyter notebook as the IDE for mechanical design. Unusual and effective.
EasyEDA Pro
Schematic & PCB layout. M5Stack H2 gateway + CoreS3 used as reference schematics.

energy

Power:
still figuring it out.

STATUS: BASELINE
LiFePO Cell ~22mAh
The boring answer. A tiny lithium iron phosphate cell similar to what Oura Ring uses. Safe, predictable, fits. The floor we build on.
STATUS: TBD
Thermoelectric Charging
Body heat differential → Peltier → trickle charge. Seductive idea. Efficiency probably terrible. Worth exploring anyway.
STATUS: TBD
Rotational Induction
The outer ring rotation drives the Hall sensor. The same magnetic pulses could theoretically power inductive charge cycles. Dual-purpose elegance.
STATUS: PLANNED
Biometric Opt-In
MAX30102 can be toggled off. Less current draw when off, your HR data stays yours when on. Two-button design philosophy.
backlog

What's
next.

Separate initialization task
Decouple setup() logic from application tasks — proper RTOS discipline.
BLE events triggered by IMU threshold
Don't flood BLE with data — only transmit when movement exceeds threshold. Better battery life.
3D → 2D projection
Map IMU 9-DoF data onto a 2D mouse movement plane. Gravity compensation, drift correction.
Mouse HID via BLE
Transition from gamepad HID to mouse HID profile. The core user-facing feature.
FlexPCB design in EasyEDA Pro
Move from reference schematics to original board layout. Constraints: must fit inner ring.
Decide on battery + charging approach
LiFePO baseline confirmed. Thermoelectric and inductive options still being evaluated.
